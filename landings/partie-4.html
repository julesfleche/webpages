01 — Le plus accessible

Claude

Avant de builder quoi que ce soit, tu dois comprendre comment parler à une IA et comment lui donner des instructions précises. C'est la base de tout.

Formation 01
Claude — Travailler avec l'IA
Asynchrone
Claude c'est ton outil principal. Pas juste pour générer du texte — pour exécuter des process complexes, prendre des décisions, interagir avec des apps via MCP. Tu dois comprendre comment il fonctionne, comment le guider, et comment lui écrire des instructions béton.
Comprendre un LLM — comment Claude raisonne, ce qu'il fait bien, ce qu'il fait mal, et pourquoi le contexte est crucial.
Prompt engineering — comment écrire un prompt précis, structuré, et reproductible. Les patterns qui fonctionnent, ceux à éviter.
Le système prompt et le contexte — comment configurer Claude pour une tâche donnée, lui donner le bon contexte, et lui donner un rôle.
L'anatomie d'un skill — structure d'un fichier SKILL.md, les sections obligatoires, comment le tester, comment l'améliorer.
Les MCPs — comment Claude se connecte à des outils externes (Notion, Slack, GitHub…) et agit dedans directement.
Skills vs Routines — la différence, quand utiliser quoi, comment enchaîner plusieurs skills avec du contexte partagé.
Cas pratique
Créer un skill OOO complet, de A à Z
Le cas pratique est directement disponible sur Skool avec toutes les instructions détaillées.

02 — La fondation

Notion

Notion c'est le système nerveux de toute la production OOO. Tu dois le maîtriser pour comprendre comment on organise le travail, suit les clients et stocke la connaissance.

Formation 02
Notion — Organiser et structurer
Asynchrone
On utilise Notion comme base opérationnelle complète : roadmaps, suivi de prod, SOPs, transcripts, to-do. Tu dois être à l'aise pour naviguer dedans, créer des pages structurées, et comprendre comment nos automatisations lisent et écrivent dans Notion.
Navigation dans le Notion OOO — comprendre l'arborescence, où est quoi, comment on organise les espaces clients et internes.
Pages et bases de données — la différence entre une page et une BDD, les types de propriétés, les vues (table, kanban, calendrier, timeline).
Relations et rollups — comment relier des bases de données entre elles et faire remonter des infos d'une BDD à une autre.
Templates OOO — les templates de portail client, de roadmap, de SOP. Comment les utiliser, les dupliquer, les adapter.
Notion API et MCP — comment n8n et Claude lisent et écrivent dans Notion. Comprendre les IDs, les propriétés, les filtres.
Cas pratique
Construire un portail client complet sur cahier des charges
Le cas pratique est directement disponible sur Skool avec toutes les instructions détaillées.

03 — Le moteur

n8n

n8n c'est là où tout s'assemble et tourne. La courbe d'apprentissage est plus raide, mais une fois que tu comprends la logique, tu peux builder n'importe quoi.

Formation 03
n8n — Automatiser et orchestrer
Asynchrone
n8n c'est le chef d'orchestre de tous nos agents. Tu vas apprendre à construire des flows robustes, à gérer les données entre les nœuds, et à connecter Claude, Notion, Slack, et toutes les APIs dont les clients ont besoin.
Interface et navigation — l'éditeur de flows, les nœuds, le panneau d'exécution, les logs, les credentials.
Les nœuds essentiels — Webhook, HTTP Request, Set, If, Code (JS), Notion, Slack, Gmail. Ceux qu'on utilise dans 80% des flows OOO.
La gestion de la donnée — comment accéder aux données d'un nœud précédent, les transformer, les filtrer, les boucler.
Connecter Claude via API — appeler Claude depuis n8n, lui passer du contexte, récupérer sa réponse et la traiter.
Les patterns OOO — les structures de flows qu'on réutilise : webhook → traitement → Notion → Slack, ou cron → Claude → livraison.
Erreurs et debugging — lire les logs, identifier où ça casse, les erreurs classiques et comment les éviter.
Cas pratique
Reproduire un flow existant, puis en builder un nouveau
Le cas pratique est directement disponible sur Skool avec toutes les instructions détaillées.
🎓 Accéder à la formation n8n sur Skool

04 — Le niveau suivant

Lovable

Lovable c'est là où on passe du système opérationnel au produit. Tu combines tout ce que tu as appris pour builder des apps complètes et les livrer à des clients.

Formation 04
Lovable — Builder des interfaces
Asynchrone
Lovable génère des apps React à partir de prompts. La compétence clé c'est de savoir comment le briefer précisément pour qu'il produise quelque chose de propre, fonctionnel, et branchable sur Supabase. Un builder OOO maîtrise Lovable parce que c'est ce qu'on livre aux clients qui ont besoin d'une interface.
Comprendre Lovable — comment il fonctionne, ce qu'il génère (React), ses limites, pourquoi on l'utilise chez OOO.
Prompter Lovable efficacement — la structure d'un bon prompt Lovable, comment décrire une interface sans ambiguïté, comment itérer proprement.
Supabase — auth et BDD — connecter Lovable à Supabase, créer les tables, configurer l'authentification, gérer les Row Level Security (RLS).
Brancher n8n en backend — utiliser n8n pour synchroniser les données Notion → Supabase, déclencher des actions depuis l'app Lovable.
Intégrer Claude dans l'app — ajouter un assistant IA contextuel dans le portail, appeler l'API Claude depuis l'interface Lovable.
Déployer et livrer — publier l'app, configurer le domaine custom, tester en conditions réelles, packager le livrable pour le client.
Cas pratique
Builder un portail client complet avec login et données live
Le cas pratique est directement disponible sur Skool avec toutes les instructions détaillées.
